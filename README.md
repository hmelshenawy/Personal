# Engineering Playbook

A personal engineering reference library.

This repository exists to help me remember, reuse, and improve what I learn while building software.

It is **not** a tutorial, a course, or a public knowledge base.

## What you'll find

* Procedures
* Guidelines
* Notes
* Templates
* Prompts
* Lessons Learned

## Core Principles

* Keep everything practical.
* Keep files short.
* Capture knowledge once.
* Reuse it many times.
* Let the structure evolve naturally.

> If I have to Google the same thing twice, it probably belongs here.
